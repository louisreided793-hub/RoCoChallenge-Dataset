RoCo Challenge:
A Dataset for Collaborative Gearbox Assembly

A real-and-simulation benchmark for human-centered robot collaboration with instruction following, error recovery, and autonomous continuation in industrial-style assembly.

Download Data Open Devkit Guide Paper / Citation

Why this dataset matters

  • Bridges sim-to-real transfer: aligned tasks and metrics across simulation and physical executions.
  • Focuses on practical failures: includes partial-state continuation and error correction, not only clean demonstrations.
  • Supports future algorithm design: useful for perception, multimodal policy learning, HRI-aware planning, and safe intervention.
  • Enables reproducible comparison: shared task definitions, scoring protocol, and baseline-ready data format.

See also: utility note for future real-data use.

What is included

Real-world subset

Bimanual robot trajectories and synchronized multimodal observations from physical collaborative assembly runs.

Simulation subset

Task-aligned simulated episodes for model development, controlled evaluation, and ablation studies.

Annotations & phases

Task-phase semantics for from-scratch assembly, partial continuation, and error-recovery behavior.

Failure/recovery cases

Scenarios with injected mistakes to evaluate correction policies and robust recovery pipelines.

Baselines

Reference setup and starter scripts for training/fine-tuning baseline policies.

Loader format, environment setup, and benchmark usage documentation.

Quick Start

  1. Download data from Hugging Face.
  2. Install dependencies (Python 3.11 and Isaac stack).
  3. Load one demonstration sequence.
  4. Run one benchmark/baseline example.

Benchmark protocol

Three tasks are scored and combined with a 4 : 2 : 4 weighting:

Final Score = (4*S1 + 2*S2 + 4*S3) / 10
TaskDefinitionScore
T1: Assembly from ScratchComplete full assembly sequence from empty state.S1
T2: Resume from Partial StateContinue correctly from a partially assembled state.S2
T3: Error Detection & RecoveryRemove wrong parts and recover to correct final assembly.S3
